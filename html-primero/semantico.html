Ir a HTML No Semántico HTML Semántico

La aplicación HTML semántica es una forma de utilizar las etiquetas de modo que cada una describa correctamente el contenido que contienen.

Cuando se habla de HTML semántico, se habla principalmente de las etiquetas. Estas etiquetas describen el rol del contenido ya sea tanto para el browser o el desarrollador y hacen que el código sea más "entendible". Por ejemplo:

  • <form>, <article> y <table> son etiquetas semánticas. Ya que, claramente indican el contenido que deberían de albergar. Algunos como <h1> o <p> son etiquetas semánticas mucho más específicas.
  • <div> y <span> no son semánticos ya que no describen el rol acerca del contenido que albergarán ni tienen un significado específico.

Un punto muy importante a tocar, es que las etiquetas HTML semánticas tienen una gran importancia para la accesibilidad. Es decir, las etiquetas HTML semánticas tienen el propósito también de hacer "accesible" el contenido para las personas con habilidades diferentes y/o discapacitadas.

Lo mostrado es un código HTML no semántico. Tan solo con verlo no se puede definir bien el contenido. Asimismo, es mucho más difícil encontrar la estructura y/o seguir el "flujo" del contenido en sí. Si bien el contenido se ve muy mal debido a que le faltan estilos. El solo hecho de agregarle estilos a los elementos los hará mucho más difíciles de leer. (A nivel de programación)

HTML Semántico

Tomando en cuenta el código anterior, utilizando correctamente las etiquetas HTML de forma semántica, el código luciría así: 

                    
    <h3>Cosas de Personajes</h3>
    <aside>
        <header>
            <h4>Facciones</h4>
        </header>            
        <ul>
            <li>The Valiant Hexen</li>
            <li>The Fried Crabs</li>
            <li>The Dusty Ones</li>
            <li>The Crazy Riders</li>
            <li>The Sweety Warriors</li>
            <li>The Naughty Rodents</li>
        </ul>
    </aside>
    <section>
        <article>
            <header>
                <h4>Armas</h4>
            </header>
            <p>
                Existe una variedad de armas que las diversas facciones utilizan. Desde aquellas que sirven para un solo 
                objetivo así como aquellas que sirven para destruir masas de individuos. ¿Serás capaz de sobrevivir ante
                estas amenazas?
            </p>
            <footer>
                <small>Tipos: a balas, explosivas, láser, químicas, mágicas.</small>
            </footer>
        </article>

        <article>
            <header>
                <h4>Escenarios</h4>
            </header>
            <p>
                Las diversas facciones viven en lugares muy curiosos y extrañamente agradables. Cualquier
                lugar puede ser utilizado como la tumba de un pobre despistado.
            </p>
            <footer>
                <small>Lugares: bosques, playas, desiertos, llanuras, montañas, etc.</small>
            </footer>
        </article>

        <article>
            <header>
                <h4>Botines</h4>
            </header>
            <p>
                Cada facción es responsable de producir elementos con los insumos y materiales que tienen a la mano.
                Sin embargo, en los momentos de escacez, ¿quién sabe?, uno mismo podría servir de mercancía.
            </p>
            <footer>
                <small>Tipos: comida, armas, reliquias, tecnología, individuos, etc.</small>
            </footer>
        </article>
    </section>

Elementos vistos

Ahora definiremos algunos de los elemento utilizados anteriormente:

  1. <aside>: consiste en un contenido indirectamente relacionado con el contenido principal.
  2. <header>: es un elemento que sirve para poder realizar encabezados.
  3. <section>: solo representa una sección genérica en la página HTML. PD: esta elemento debería de tener siempre un encabezado.
  4. <article>: es parecido a una sección, pero su contenido debe de ser independiente y tener sentido por sí mismo.
  5. <footer>: simplemente representa un pie de página, sección, artículo, etc.
  6. <small>: este elemento puede albergar información complementaria como comentarios o notas explicativas. La fuente de su contenido es pequeña por defecto.
